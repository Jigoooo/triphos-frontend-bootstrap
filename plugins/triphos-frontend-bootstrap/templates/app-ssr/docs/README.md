# Triphos Frontend System Of Record

This generated frontend repository treats `docs/` as the canonical source for:

- architecture and harness surfaces
- product intent and scope
- tracked plans and verification status
- decisions that should outlive prompt history
- quality gates and visual baselines

Read these in order:

1. `docs/product/README.md`
2. `docs/architecture/README.md`
3. `docs/quality/verification-matrix.md`
4. `docs/plans/README.md`
5. `docs/decisions/README.md`

Important defaults:

- This strong harness is intended for repositories generated by `triphos-frontend-init`.
- Existing projects only adopt this harness through an explicit `triphos-frontend-adopt` migration.
- `AGENTS.md` and `CLAUDE.md` are entrypoints. `docs/` is the durable record.
