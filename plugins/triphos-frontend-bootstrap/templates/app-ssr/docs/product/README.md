# Product Intent

This generated app is the Triphos frontend bootstrap baseline for company projects.

## Scope

- The harness is strong by default for repositories created with `triphos-frontend-init`.
- `triphos-frontend-adopt` is the explicit migration path for an existing frontend project that should opt into the same rules.
- Repositories that were not generated or explicitly adopted should not be assumed to follow this contract.

## Success criteria

- Agents can discover intent, decisions, and verification requirements from repo-local files.
- Non-trivial work leaves durable plan, decision, and verification artifacts in `docs/plans/`.
- Frontend verification proves runtime behavior, not only lint/typecheck compliance.
